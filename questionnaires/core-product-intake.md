# Questionnaire: Core Product Intake

Use this intake before running most flows.

Ask the user these questions in a structured way. If they do not know an answer, mark it as unknown rather than guessing.

## Core Questions

- Project or system name
- Organisation type and sector
- Jurisdiction or market of deployment
- Intended purpose of the AI system
- Main users of the system
- Individuals or groups affected by the outputs
- Organisation role: provider, deployer, importer, distributor, controller, processor, or mixed
- Current phase: idea, procurement, pilot, internal use, external deployment, live production
- Data categories used
- Whether special-category or criminal-offence data is involved
- Main decisions or actions influenced by the system
- Whether outputs could have legal or similarly significant effects on individuals
- Known vendors, model providers, or subprocessors involved
- Deployment timeline
- Known constraints, risks, or unresolved concerns

## Follow-Up Rule

If the user gives short or vague answers, ask for:

- one concrete example input
- one concrete example output
- who is most affected if the system is wrong
- whether a human can override the output in practice
