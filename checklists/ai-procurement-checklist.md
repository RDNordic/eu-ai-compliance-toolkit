# AI Procurement Checklist

Use this checklist before procuring an external AI tool, model API, or AI-enabled SaaS product.

## Scope

- [ ] Intended business use is clearly described.
- [ ] Affected users and individuals are identified.
- [ ] The organisation role is understood at a high level.

## Vendor Transparency

- [ ] Vendor and product name recorded.
- [ ] Core functionality is described in plain language.
- [ ] Model provider chain is identified where possible.
- [ ] Processing locations are disclosed.
- [ ] Subprocessors are disclosed or requested.

## Privacy And Data Handling

- [ ] Expected input data categories are listed.
- [ ] Personal-data use is identified.
- [ ] Special-category or sensitive data exposure is assessed.
- [ ] Retention and deletion controls are documented.
- [ ] Training-on-customer-data position is documented.
- [ ] DPA or contractual privacy terms are reviewed.

## Security And Access

- [ ] Access-control model is described.
- [ ] Admin or audit features are documented.
- [ ] Security documentation has been requested or reviewed.
- [ ] Support-access or human-review access paths are understood.

## Compliance And Governance

- [ ] AI Act classification screen is completed.
- [ ] GDPR lawful basis implications are assessed.
- [ ] DPIA screen is completed where relevant.
- [ ] Claim-source register or equivalent evidence record is updated.
- [ ] Legal or privacy escalation is logged for unresolved points.

## Decision

- [ ] Go
- [ ] Conditional go
- [ ] No-go

## Notes

Record missing answers, contract dependencies, and next actions before procurement approval.
