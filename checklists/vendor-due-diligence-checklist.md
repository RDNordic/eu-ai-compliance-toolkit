# Vendor Due Diligence Checklist

Use this checklist when assessing an AI vendor or AI-enabled supplier.

## Product And Supplier Basics

- [ ] Vendor legal entity is recorded.
- [ ] Product or service name is recorded.
- [ ] Intended use within the organisation is defined.
- [ ] Critical dependency on the vendor is understood.

## Data Questions

- [ ] What data will users input
- [ ] Whether personal data is expected
- [ ] Whether special-category or confidential data may appear
- [ ] Whether prompts, files, or outputs are retained
- [ ] Whether prompts, files, or outputs are used for training
- [ ] Whether customer deletion controls exist

## Governance Questions

- [ ] DPA or equivalent contractual terms are available.
- [ ] Subprocessors are disclosed.
- [ ] Processing locations are disclosed.
- [ ] Breach or incident notification terms are reviewed.
- [ ] Customer configuration controls are documented.

## Technical And Security Questions

- [ ] Authentication and access controls are reviewed.
- [ ] Admin visibility and audit options are reviewed.
- [ ] Vendor security materials are requested.
- [ ] Support-access model is understood.

## Use-Case Suitability

- [ ] The tool’s outputs are not being over-trusted for high-impact decisions.
- [ ] Human review requirements are clear.
- [ ] Known limitations are acceptable for the use case.
- [ ] Escalation is triggered for unresolved material issues.

## Recommendation

- [ ] Go
- [ ] Conditional go
- [ ] No-go
