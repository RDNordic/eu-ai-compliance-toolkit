# Safe Usage

Use this repository as if it were a disciplined operating kit, not a magic compliance button.

## Recommended Safe Workflow

1. Start with a generic or redacted fact pattern.
2. Run the relevant questionnaire and workflow.
3. See what facts are actually needed.
4. Add only the minimum extra detail required.
5. Escalate to legal, privacy, or security review when the issue is material.

## Good Inputs

- high-level system descriptions
- anonymised use cases
- vendor documentation summaries
- deployment assumptions
- role and market information

## Inputs Requiring Caution

- personal data
- special-category data
- confidential contracts
- internal security architecture
- source code from sensitive systems
- procurement documents with non-public terms

## User Rule

If you would hesitate to paste the material into a third-party service, stop and review your AI tool’s contractual and technical controls before continuing.
