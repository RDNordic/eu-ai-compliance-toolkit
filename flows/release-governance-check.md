# Flow: Release Governance Check

Use this flow before internal release, public launch, pilot expansion, or procurement sign-off.

## Use When

- a team says the product is “basically ready”
- governance evidence needs a final pass
- the project needs a concise go / no-go record

## Inputs To Collect

Review:

- [`../starter-pack/compliance-gate.md`](../starter-pack/compliance-gate.md)
- outputs from other flows in this repo

Collect any missing facts on:

- unresolved legal questions
- risk owner
- deployment timing
- monitoring plan
- documentation completeness

## Required Output

Produce:

- [`../outputs/compliance-decision-record.md`](../outputs/compliance-decision-record.md)

Optionally append:

- [`../outputs/legal-escalation-note.md`](../outputs/legal-escalation-note.md)

## Output Must Include

- release objective
- gating issues
- unresolved blockers
- owner assignments where known
- go / conditional go / no-go recommendation
- confidence level
- immediate next action
