# Next Test Steps

Use this file as a lightweight action list for the first external trial of the repo.

## Goal

Get a trusted external user to run the repo on a real company use case in a safe, controlled way and return practical feedback.

## Proposed Tester

- Chris from NMI AI

## Safe Trial Plan

1. Ask Chris to test the repo on a real but non-sensitive internal use case.
2. Tell him to start with:
   - [`START-HERE.md`](START-HERE.md)
   - [`trust/PRIVACY.md`](trust/PRIVACY.md)
   - [`trust/SAFE-USAGE.md`](trust/SAFE-USAGE.md)
3. Ask him to avoid:
   - direct personal data
   - confidential contracts
   - secrets
   - production-only security details
4. Ask him to use redacted or summarized facts where possible.
5. Ask him to run one full path end-to-end:
   - questionnaire
   - flow
   - output template
6. Ask him to note where the repo felt:
   - useful
   - confusing
   - too legal
   - too vague
   - missing a practical step

## Feedback Questions For Chris

- What was the first thing you tried to do?
- Did the repo make that next step obvious?
- Did the assistant ask the right questions?
- Did the output feel usable for a real team?
- Where did trust, privacy, or security concerns come up?
- What would stop you from sharing this with others?
- What would make this feel production-ready?

## Deliverable From Trial

Capture:

- use case tested
- workflow used
- output generated
- trust or privacy concerns raised
- friction points
- suggested improvements

## Immediate Follow-Up

After Chris tests it:

1. Fix the biggest confusion point.
2. Turn his use case into a worked example if he agrees.
3. Use that example to sharpen the README and launch messaging.
